================================================================================
1. USER LANGUAGE SYNTAX
================================================================================

Expr :=
    Atom
  | (defun Name (Args*) Body)
  | (fn (Args*) Body)
  | (def Name Expr)
  | (let (Var Expr) Body)
  | (if Cond Then Else)
  | (try Expr (catch Var Handler))
  | (assert Condition Msg)
  | (ffi-call Fn Args Timeout)
  | (ffi-pin Expr)
  | (pre Condition Expr)
  | (post Condition Expr)
  | (unsafe Expr)

Evaluation:
- Strict left-to-right evaluation
- Pure Lisp syntax (minimal surface forms)
- def, fn, defun are aliases

================================================================================
2. TYPE SYSTEM
================================================================================

Type :=
    Int | Float | Bool | String | Unit
  | Vec<T> | Map<K,V>
  | TCap<T>        // immutable capability
  | TMut<T>        // exclusive mutable capability
  | TAtomic<T>
  | TBox<T>        // heap-managed
  | TWeak<T>
  | TFun([T...], T)
  | TPin<T>

Rules:
- TMut: exclusive ownership
- TCap: shared immutable access
- TBox: GC-managed or region-managed heap object
- TPin: FFI-safe memory-only immutable exposure

================================================================================

================================================================================
6. FFI MODEL
================================================================================

ffi-call(Fn, Args, Timeout)

Rules:
- timeout enforced
- calls must satisfy FFI_Pinnable constraints if pinned
- no OS handles in pinned memory
- violations trigger rollback

ffi-pin:
- only memory-safe types allowed
- immutable view into foreign-accessible memory
